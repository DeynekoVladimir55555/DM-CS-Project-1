# DM-CS-Project-1
Create a computer algebra system. 

[Ссылка на таблицу с модулями](https://docs.google.com/document/d/1TNSF2cwcd-7z0w6qAfidsigiQliIXhzCgGgf0g6X1Yw/edit?tab=t.0)

**Классы**

Для работы с данными были реализованы 4 класса: NaturalNumber, IntNumber, RationalNumber, Polinom.
Они хранят данные и имеют методы для выполнения задач.

В NaturalNumber хранятся массив цифр в обратном порядке(начиная с единиц заканчивая самым старшим разрядом) и индекс последней цифры.

В IntNumber помимо тех же полей что и в NaturalNumber, хранится знак числа, принимающий значение 1 если плюс, -1 если минус и 0 если число равно 0.

В RationalNumber хранятся числитель в виде IntNumber и знаменатель в виде NaturalNumber.

В Polinom хранится его степень, а также коэффициенты в виде словаря, где по степени Х(ключу) хранится соответствующий коэффициент в виде RationalNumber.

**Методы классов**

**NaturalNumber** — натуральные числа, хранящиеся в виде массива цифр в обратном порядке.

`nzer_n_b` проверяет, является ли число нулём, сравнивая массив цифр с `[0]`.

`add_1n_n` прибавляет единицу, распространяя перенос по разрядам и при необходимости добавляя старший разряд.

`mul_nd_n` умножает число на однозначную цифру, проходя по разрядам с накоплением переноса.

`mul_nk_n` умножает на 10^k, дописывая k нулей в начало массива цифр.

`mul_nn_n` умножает два числа через последовательное сложение произведений каждой цифры второго числа на первое, сдвинутых на соответствующий разряд.

`add_nn_n` складывает два числа поразрядно с переносом, выравнивая длины массивов.

`sub_nn_n` вычитает меньшее из большего поразрядно с заимствованием, определяя порядок операндов через сравнение.

`com_nn_d` сравнивает два числа — сначала по длине, затем поразрядно со старшего разряда, возвращая 0, 1 или 2.

`div_nn_n` делит нацело через итеративное нахождение цифр частного с помощью `div_nn_dk` и вычитания соответствующих кратных делителя.

`mod_nn_n` находит остаток как разность делимого и произведения делителя на частное.

`div_nn_dk` находит старшую цифру частного и её позицию k, подбирая наибольшее count такое, что `count * number * 10^k <= self`.

`sub_ndn_n` вычитает из числа произведение `number * digit`, предварительно вычислив его и проверив допустимость операции.

`gcf_nn_n` находит НОД алгоритмом Евклида через последовательное взятие остатка.

`lcm_nn_n` находит НОК через формулу `a * b / НОД(a, b)`.

---

**IntNumber** — целые числа со знаком (`-1`, `0`, `1`) поверх натуральных.

`sgn_z_d` возвращает знак числа.

`abs_z_z` возвращает абсолютную величину, меняя знак на положительный если он отрицательный.

`mul_zm_z` меняет знак числа на противоположный умножением на -1.

`trans_n_z` преобразует натуральное в целое, присваивая знак `1`.

`trans_z_n` преобразует целое неотрицательное в натуральное через строковое представление.

`add_zz_z` складывает два целых: если знаки одинаковы — складывает модули, иначе вычитает меньший из большего и берёт знак большего.

`sub_zz_z` вычитает через сложение с противоположным: `a - b = a + (-b)`.

`mul_zz_z` перемножает модули через `mul_nn_n`, знак результата определяется по правилу знаков.

`div_zz_z` делит модули через `div_nn_n`, знак определяется по правилу знаков, ноль остаётся нулём.

`mod_zz_z` находит остаток как `self - (self / other) * other`.

---

**RationalNumber** — рациональные числа в виде пары `IntNumber` (числитель) и `NaturalNumber` (знаменатель).

`red_q_q` сокращает дробь: находит НОД числителя и знаменателя и делит оба на него, возвращая новый объект.

`int_q_b` проверяет, является ли дробь целым числом, сравнивая знаменатель с `"1"`.

`trans_z_q` преобразует целое в дробное с знаменателем `1`.

`trans_q_z` преобразует сокращённую дробь с знаменателем `1` в целое.

`mul_qq_q` перемножает числители и знаменатели двух дробей независимо, возвращая несокращённый результат.

`div_qq_q` делит дроби через умножение на обратную: числитель и знаменатель делителя меняются местами.

`add_qq_q` приводит дроби к общему знаменателю через НОК, складывает числители; нулевые слагаемые обрабатываются отдельно.

`sub_qq_q` аналогично сложению, но вычитает числители; если уменьшаемое ноль — меняет знак вычитаемого.

---

**Polinom** — многочлен, хранящийся как словарь `{степень: RationalNumber}`.

`deg_p_n` возвращает степень многочлена.

`led_p_q` возвращает старший коэффициент.

`mul_pxk_p` умножает на x^k сдвигом всех ключей словаря на k, возвращает новый объект.

`mul_pq_p` умножает каждый коэффициент на рациональное число через `mul_qq_q`, возвращает новый объект.

`der_p_p` дифференцирует: каждый коэффициент при x^i умножается на i и переносится на степень i-1.

`fac_p_q` вычисляет НОД числителей и НОК знаменателей всех коэффициентов, возвращая общий множитель как рациональное число; знак определяется чётностью количества отрицательных коэффициентов.

`add_pp_p` складывает коэффициенты при одинаковых степенях через `add_qq_q`, недостающие степени заменяются нулём.

`sub_pp_p` вычитает коэффициенты при одинаковых степенях через `sub_qq_q`, убирая нулевые старшие члены.

`mul_pp_p` перемножает многочлены классически: каждый коэффициент первого умножается на каждый коэффициент второго, результаты накапливаются по степеням.

`div_pp_p` делит с остатком: итеративно вычитает из делимого произведение делителя на очередной член частного, определяя его через деление старших коэффициентов.

`mod_pp_p` находит остаток как `self - делитель * частное`, используя `div_pp_p` и `mul_pp_p`.

`gcf_pp_p` находит НОД двух многочленов алгоритмом Евклида через `mod_pp_p`.

`nmr_p_p` устраняет кратные корни делением на НОД многочлена и его производной: `P(x) / НОД(P(x), P'(x))`.

**Интерфейс**

Интерфейс состоит из 3 окон. Главное с полями ввода и вывода, окно справки с информацией о программе и отдельное для вывода многочленов.

Переключение между блоками задач осуществляется с помощью правого блока кнопок, выбор конкретной задачи с использованием выпадающего списка.

После ввода данных пользователем в соответствующие поля и выбором операции, которую необходимо произвести нажатием кнопки запускается обработка выбранной задачи.

Программа проверит валидность данных, затем попытается выполнить задачу, после чего выведет ответ в определённое поле. Если же выполнить задачу не удалось, программа выведет соответствующее сообщение об ошибке.
